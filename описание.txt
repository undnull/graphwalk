---НА ЧЕМ НАПИСАНО---
Версия языка: C# 10.0
Версия CLR: 6.0.5

---ФАЙЛЫ---
Проект: HW.csproj
Исходник: Program.cs
Файл с входными данными: input.txt
Файл с выходными данными: output.txt
Граф с практики: pract.txt

---КАК ЗАПУСКАЛОСЬ---
Из POSIX-совместимой оболочки:
    dotnet run < input.txt > output.txt
Для графа с практики:
    dotnet run < pract.txt

---ФОРМАТ ВВОДА---
Сначала вводятся пары чисел,
представляющие рёбра графа, по каждому
ребру на строку. Затем вводится "done".
После чего вводится вершина, с которой
необходимо начинать работу.

---ФОРМАТ ВЫВОДА---
1.  В самом начале выходного файла находятся входные
    значения ребер, которые распарсила программа.
2.  Начальная вершина.
3.  Данные о проходе через каждую вершину, а также
    сообщения о "коллапсе" вершины или ребра. Про
    "коллапсирование" в моём алгоритме я поясню на паре.
4.  Данные об обратных ребрах (ребро, сколлапсировавшее
    в сторону вершины, являющейся в некой степени предком
    текущей обрабатываемой). Данные ребра на практике
    помечались циановым цветом.
5.  Полученные точки сочленения. За их правильность я
    не ручаюсь говорить, однако на глаз неправильной
    является только одна (9, которая, очевидно, сочленением
    для варианта номер 4 не является).
